# Step 3 Prompt: Produce Revised Script Draft

你现在执行的是「剧本洗稿」流程的第 3 步：输出洗稿后剧本草稿。

你会收到：

- 规范化原稿
- 洗稿方案或共享刷新圣经

你的任务是严格依据方案完成洗稿重写，输出可供最终质检的完整剧本草稿。

## 核心要求

1. 严格执行洗稿方案中的替换与规避规则。
1.1 如果输入中存在「共享刷新圣经」，它就是整个项目的唯一规范来源，优先级高于临时判断。
2. 保留原有剧情逻辑、结构顺序、戏剧功能、节奏关系。
3. 对白、场景说明、人物称呼、关键道具写法都要统一。
4. 敏感词替换必须自然，不能导致语义断裂。
5. 输出应当像完整可读的剧本，而不是替换演示稿。
6. 默认是中文项目：
   - 人物、组织、地点、系统、能力、法宝、头衔都应采用刷新后的规范中文名
   - 不要输出英文音译主名
   - 不要输出 `新名（旧名）`、`Yu Xiao（余晓）` 之类校对型混合写法，除非明确进入 audit 模式

## 输出目标

- 直接输出完整洗稿后剧本草稿
- 不要加解释
- 不要附加“以下是修改说明”
- 不要再输出方案摘要

## 质量标准

- 读起来通顺自然
- 替换前后逻辑一致
- 人物关系不混乱
- 关键名词前后一致
- 平台风险词尽量规避
- 刷新圣经中的世界观专有名词、系统名、契约名、能力名、法宝名、势力名都要统一执行

## 禁止事项

- 不要缩写剧情
- 不要删除关键场景
- 不要把剧本改成梗概或大纲
- 不要输出最终 QA 说明

只输出洗稿后剧本草稿正文。

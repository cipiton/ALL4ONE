# Step 1 Prompt: Read And Normalize Source

你现在执行的是「剧本洗稿」流程的第 1 步：读取并规范原稿。

你的任务不是改写成最终洗稿稿件，而是先把原始文本整理成后续步骤可稳定使用的“规范化原稿包”。

## 输入目标

你会收到原始剧本或原始文本。必须完整阅读，并在不改变剧情主线的前提下完成规范化整理。

## 核心要求

1. 完整通读原文，不得跳读。
2. 不进行最终洗稿，不提前执行第 2 步或第 3 步。
3. 不改写剧情走向，不新增人物或情节。
4. 可以修正明显的格式噪音、称谓混乱、段落断裂、同一对象多种写法等问题。
5. 保留剧本结构感，包括场景、旁白、对白、角色出场关系。
6. 如果同时提供了「共享刷新圣经」，必须把它当作项目级规范来源：
   - 默认使用中文刷新后的规范称呼
   - 不要把中文名转成英文或拼音主名
   - 不要输出 `新名（旧名）` 这种最终交付不需要的混合写法

## 输出格式

按以下结构输出纯文本：

### 原稿概况
- 文本类型
- 题材/风格
- 主要矛盾
- 主要人物
- 是否存在明显敏感内容或命名混乱

### 统一命名与术语基线
- 列出当前原稿里应当视为同一对象的不同叫法
- 给出建议统一后的写法
- 只建立基线，不做最终替换方案展开
- 如果共享刷新圣经已给出规范名，必须优先沿用，不得另起新名

### 规范化原稿
- 输出整理后的全文
- 保持原有结构与剧情顺序
- 修正明显格式问题
- 保留所有关键剧情信息
- 不要在正文里混入英文音译主名或中英混排主名

## 禁止事项

- 不要输出洗稿方案
- 不要输出替换表大纲之外的大量建议名
- 不要输出最终洗稿稿件
- 不要写成摘要

最终输出必须是可供第 2 步继续处理的规范化原稿包。

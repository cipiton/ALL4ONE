# 步骤三：输出生图配置

## 执行前置条件
- 用户已确认步骤二的资产清单和详情
- 用户已在步骤二中选择了资产风格（写实/2D/3D）
- 基于步骤二的资产信息生成txt格式文档
- 直接输出txt文档内容

## 生图配置文档规范

### 文件格式
- 输出格式：纯文本（txt）
- 编码格式：UTF-8
- 无需特殊格式，纯文本即可

### 文档结构
按照以下顺序组织内容：
1. 角色配置
2. 场景配置
3. 道具配置

每个大类内部按序号排列。

### 风格与光线说明
根据步骤二用户选择的风格，应用对应的风格及光线描述：
- **写实风格**：照片级写实，电影质感，柔和自然光
- **2D风格**：2D AI漫剧风格，光影质感细腻、层次丰富
- **3D风格**：高精度3D CG风格，光影质感真实细腻，柔和自然光

## 具体格式要求

### 1. 角色配置格式

```
角色
[序号]
[角色名称]
[生图提示词内容]

[角色名称]音色
[音色说明内容]
```

**说明**：
- 每个角色占用4-5行（角色名称、核心特征、生图提示词，然后是音色说明）
- 核心特征描述角色的核心视觉特征（必须为视觉类特征，严禁包含身份类、职业类、地位类、性格类等非视觉特征）
- 核心特征允许包含：衣服颜色/款式、年龄段、身体特征（发型、胡须、体型等）、性别
- 核心特征禁止包含：
  - 身份类：皇帝、主角、反派、将军、老大等
  - 职业类：士兵、商人、医生、老师等
  - 地位类：老大、小弟、上司等
  - 性格类：冷酷、威严、温柔、可爱、暴躁等
- 核心特征字数限制：不超过10字
- 生图提示词包括：风格及光线、输出要求、主体内容
- 音色说明包括：性别声、音域、音色、语速、seed

### 2. 场景配置格式

```
场景
[序号]
[场景名称]
[生图提示词内容]
```

**说明**：
- 每个场景占用4行（场景名称后跟随生图提示词）
- 生图提示词包括：风格及光线、输出要求、主体内容

### 3. 道具配置格式

```
道具
[序号]
[道具名称]
[生图提示词内容]
```

**说明**：
- 每个道具占用4行（道具名称后跟随生图提示词）
- 生图提示词包括：风格及光线、输出要求、主体内容
- 道具的特殊要求：2-4不同角度展示并标注角度

## 生图提示词模板

### 角色生图提示词模板
```
核心特征：[不超过10字的核心视觉特征]
风格及光线：照片级写实，电影质感，柔和自然光
输出要求：高清4k，细节丰富，纯白背景，头部正面特写布局在左边，正面/侧面/背面全身三视图布局在右边，全部集中在一张图片输出
主体内容：[年龄、脸型、发型、身材、服饰等描述，严禁包含表情描述]
```

**主体内容描述要求**
- 专注于角色静态外貌特征
- 包含：年龄、脸型、发型、身材、服饰、身体特征等
- **严禁包含表情描述**：如"眼神锐利"、"表情冷酷"、"微笑"、"愤怒"等
- 示例：
  - ✅ 正确：45岁左右的男性，威严的龙相国字脸，浓密的黑色胡须，身材健硕挺拔，穿着明黄色龙袍，头戴翼善冠
  - ❌ 错误：45岁左右的男性，威严的龙相国字脸，眼神锐利如鹰，浓密的黑色胡须，身材健硕挺拔，穿着明黄色龙袍，头戴翼善冠（包含"眼神锐利"表情描述）

### 场景生图提示词模板
```
风格及光线：照片级写实，电影质感，[具体光线描述]
输出要求：高清4k，细节丰富
主体内容：[环境、建筑、装饰、氛围等描述，严禁包含具体角色]
```

**主体内容描述要求**
- 专注于场景本身的描述
- 包含：环境、建筑、装饰、氛围等
- **严禁包含具体角色**：如"李默坐在"、"朱棣站在"、"赵虎走过"等
- 可以描述路人、群众等非特定角色背景：如"熙熙攘攘的人群"、"路过的行人"等
- 示例：
  - ✅ 正确：热闹的火锅店大厅，中式传统风格装修，红色灯笼点缀，木质桌椅整齐排列，摆放着火锅、调料罐和餐具，充满烟火气但气氛紧张
  - ❌ 错误：热闹的火锅店大厅，李默坐在窗边，中式传统风格装修，红色灯笼点缀，木质桌椅整齐排列（包含具体角色"李默"）

### 道具生图提示词模板
```
风格及光线：照片级写实，电影质感，柔和自然光
输出要求：高清4k，细节丰富，背景纯白，2-4不同角度展示并标注好角度，集中放在同一张图输出
主体内容：[材质、大小、颜色、功能等描述]
```

## 执行注意事项

### 与步骤二的一致性
- 角色名称必须与步骤二的资产总清单完全一致
- 生图提示词内容必须与步骤二的详情完全一致
- 音色说明必须包含seed参数

### 序号规范
- 每个大类（角色/场景/道具）单独编号
- 序号从1开始，连续排列
- 无需跳号或留空

### 换行处理
- 各大类之间空一行
- 同一类别内的不同资产之间**不空行**，紧密排列
- 生图提示词的三个部分（风格及光线、输出要求、主体内容）各占一行

### 特殊要求处理
- 道具必须包含"2-4不同角度展示并标注好角度"的要求
- 角色必须包含"全身构图，平视机位，纯白背景"的要求
- 场景和角色的"风格及光线"可根据实际情况调整

### 空资产处理
- 如果某个类别没有资产（如没有道具），则不输出该类别
- 仅输出实际存在的资产类别

### 文本纯净性
- 不添加任何额外的标题、说明或注释
- 仅输出纯粹的配置内容
- 确保文本可直接用于AI生图工具

## 输出完成标志

完成txt文档生成后，明确告知用户：
- 已完成生图配置文档输出
- 文档包含角色X个，场景Y个，道具Z个
- 文档格式为txt，可直接用于AI文生图工具
- 无需等待确认（步骤三直接输出文档）

# 步骤一：输出解说剧剧本

## 执行前置条件

### 情况A：用户从零开始创作解说剧
在开始剧本创作前，必须先询问用户期望的剧本集数。若用户设定的集数明显不合理，需给予反馈或建议。

### 情况B：用户提供影视剧分集剧本要求改写成解说剧剧本
当用户提供的是影视剧分集剧本时，必须：
- **保持剧情内容不变**：包括前置钩子、核心剧情、结尾悬念等所有情节
- **保持集数不变**：不增减集数
- **仅改为旁白形式**：将原剧本转换为以旁白解说为主的形式
- **优先使用主角第一人称**：以主角第一人称独白视角推进剧情

### 集数合理性评估标准（仅情况A）
- 单集字数应控制在600-1200字之间
- 集数过少（如1-2集）可能导致剧情压缩过紧
- 集数过多（如10集以上）可能导致剧情松散
- 建议集数：3-6集为最佳范围

## 剧本创作核心要求

### 1. 旁白与对白比重

**旁白为主（占比≥70%）**
- 主要由旁白解说推动剧情发展
- 画面配合旁白呈现
- 信息密度高，符合短剧观看习惯

**对白为辅（占比≤30%）**
- 仅在关键情节处保留人物对白
- 对白用于强化情感冲突或重要信息传递
- 避免冗长对话，每段对白不超过3句

**旁白视角选择**
- 优先采用主角第一人称独白
- 根据剧情需要在男主/女主视角间切换
- 若男女主独白均不合适，使用第三人称旁白

### 2. 内容原则

**符合解说剧节奏**
- 信息密度高，每句话都有推进剧情或塑造人物的作用
- 符合短剧快节奏、强悬念的观看习惯
- 避免长篇铺垫，快速进入核心冲突

**分集结构规范**
每集剧本必须包含三个部分：
- **前置钩子**：最精彩、最有悬念、最吸引人的设定或情绪最强烈的情节
- **核心剧情**：发展主线情节，推动故事进展
- **结尾悬念**：留下悬念，引发观众期待下一集

**情况B特殊要求（改写影视剧剧本）**
- **严格保持原剧本的结构和内容**：前置钩子、核心剧情、结尾悬念必须与原剧本一致
- **不修改剧情发展**：不改变任何情节走向、事件顺序和关键转折
- **仅改变叙述方式**：将原剧本中的对白、动作、场景描述转换为旁白形式
- **视角统一**：优先以主角第一人称视角叙述，将原剧本中的客观描述改为"我看到"、"我听到"、"我感觉"等主观视角
- **对白保留原则**：仅保留对塑造人物或推进剧情至关重要的简短对白（不超过原对白的30%）

**第一集特别要求**
- 必须将全剧最精彩、最吸引人的设定放在最前面
- 快速建立主角困境和核心冲突
- 立即展示钩子，引发观众兴趣

**后续分集钩子策略**
- 以上一集剧情回顾作为前置钩子（适用于连续剧情）
- 承接上一集的结尾悬念作为前置钩子（适用于悬念递进）
- 或展示本集核心冲突的高光时刻（适用于独立成集）

### 3. 时长与字数控制

**时长规范**
- 每集时长：1-2分钟
- 每集字数：600-1200字
- 按正常解说剧语速计算

**镜头时长原则**
- 每个镜头时长必须大于旁白/台词时长
- 确保观众有足够时间理解画面内容
- 避免画面切换过快导致信息遗漏

## 剧本输出格式

### 分集输出模板

```
第X集
前置钩子：【钩子内容，30-80字】

核心剧情：【核心剧情内容，500-900字】

结尾悬念：【悬念内容，50-150字】
```

### 内容要求
- 前置钩子：简洁有力，立即抓住观众注意力
- 核心剧情：详实完整，展现故事发展脉络
- 结尾悬念：引人遐想，激发继续观看欲望

## 参考示例

### 第1集示例
```
第1集
前置钩子：32岁打工仔混到除夕夜，负债累累躲在出租屋连家都不敢回，亲妈一通催泪电话，居然让我撞大运激活了情绪换钱的神级系统？这波是绝境逢生还是雪上加霜？

核心剧情：我叫江辰，32岁，在大城市摸爬滚打多年，没房没车没存款还欠着一屁股债，除夕夜缩在10平米出租屋，连碗热饺子都吃不上，满脑子都是还债和面对爸妈的压力。窗外鞭炮声呛人，我妈的电话带着哭腔："辰啊，今年真不回了？妈不盼你赚钱，就想看看你。"这话扎得我愧疚、委屈、不甘全涌上来，攥着手机的手都在抖。就在这时，脑子里"叮"的一声，机械音炸响：【心潮兑换系统已绑定！他人对宿主的真实情绪可转化为情绪值，1情绪值=10元人民币，单日提现上限1万元】。我掐了自己一把，不是梦！盯着手机里30万的网贷额度，我心想烂泥塘里总得赌一把，靠这系统清债、风风光光回村，这30万就是我的翻身本钱。

结尾悬念：这天上掉的系统到底靠不靠谱？我刷爆30万网贷孤注一掷，真能靠别人的情绪赚大钱，从负债屌丝逆袭成人上人吗？
```

### 第2集示例
```
第2集
前置钩子：上一集结尾，江辰刚刚激活心潮兑换系统，决定豪赌30万网贷，这荒唐的决定让他一夜之间成了全村的笑柄，还是迎来了人生的第一个转机？

核心剧情：第二天一早，我顶着黑眼圈就冲向了人群最密集的菜市场。系统提示显示，越是真实强烈的情绪，兑换的情绪值越高。我穿着破旧夹克，蹲在菜摊前观察，一个小贩因为顾客砍价涨红了脸，情绪值+50。我试着上前搭话，故意激怒他，他果然暴跳如雷，情绪值+200！我震惊了，这来钱也太快了吧？正当我得意忘形时，一个熟悉的身影出现在人群里——那是我的前女友小雅，她挽着一个西装革履的男人，谈笑风生。看到她的瞬间，我心如刀绞，系统提示：情绪值+500。原来负面情绪也能兑换？我愣在原地，小雅发现了我的狼狈，眼神里闪过一丝不屑。

结尾悬念：前女友的出现让我情绪崩溃，系统居然疯狂收割我的负面情绪值？这个系统到底是在帮我，还是在玩我？
```

## 执行注意事项

### 旁白视角一致性
- 同一集内保持旁白视角统一
- 如需切换视角，在旁白中明确标注（如："江辰的视角……"）
- 避免频繁切换导致观众困惑

### 情况B特别注意事项（改写影视剧剧本）
- **严禁修改剧情内容**：前置钩子、核心剧情、结尾悬念必须与原剧本完全一致
- **严禁增减集数**：保持原剧本的集数不变
- **严格转换叙述视角**：将原剧本的客观叙述改为主角第一人称视角
  - 原剧本："江辰走进菜市场，看到小贩正在和顾客争吵"
  - 解说剧剧本："我走进菜市场，看到小贩正在和顾客争吵"
- **对白处理原则**：
  - 将大部分对白改为旁白叙述（如："他对我说……"）
  - 仅保留最具代表性的简短对白（不超过原对白的30%）
  - 保留的对白必须能直接展现人物性格或推动关键情节
- **场景转换**：将原剧本中的场景描述改为旁白中的场景交代
- **动作描述**：将原剧本中的动作描写改为旁白中的叙述性描述

### 信息密度控制
- 每句话都必须有信息增量
- 删减冗余描述和无关细节
- 确保旁白简洁有力，不拖沓

### 悬念设置技巧
- 结尾悬念要足够吸引人，但不能剧透后续剧情
- 悬念要与下一集的前置钩子形成呼应
- 避免悬念设置过于牵强或突兀

### 剧情连贯性
- 各分集剧情要形成完整的故事线
- 确保核心冲突持续发展
- 伏笔和呼应要清晰明确

## 输出完成标志

完成剧本创作后，明确告知用户：
- 已完成分集剧本输出，共X集
- 询问用户是否需要调整或修改
- 等待用户确认后，根据用户要求决定是否输出文档

# 步骤二：提炼资产

## 执行前置条件
- 用户已确认步骤一的剧本内容，或用户直接提供了剧本内容
- 如果共享运行时尚未提供 `style`，先询问用户选择资产风格（写实/2D/3D）
- 如果共享运行时已经提供 `style`，直接按该风格提炼资产，不要再次询问、不要等待确认
- 基于用户选择的风格和剧本内容进行资产提炼
- 按照规范输出资产总清单和资产详情

## 资产风格选择（必须先确认）

仅在共享运行时没有提供 `style` 时，才需要询问用户选择生图风格：

### 风格选项

**1. 写实风格**
- 风格及光线：照片级写实，电影质感，柔和自然光
- 适用场景：现代都市、悬疑、现实主义题材
- 特点：高度还原真实世界，细节丰富

**2. 2D风格**
- 风格及光线：2D AI漫剧风格，光影质感细腻、层次丰富
- 适用场景：动漫、二次元、青春校园题材
- 特点：扁平化设计，色彩鲜艳，线条清晰

**3. 3D风格**
- 风格及光线：高精度3D CG风格，光影质感真实细腻，柔和自然光
- 适用场景：科幻、奇幻、大型IP改编题材
- 特点：立体感强，材质逼真，光影效果出色

### 确认流程
1. 向用户展示三种风格选项及特点
2. 询问用户选择哪种风格
3. 根据用户选择确定所有资产的风格及光线描述
4. 开始资产提炼流程

## 资产提炼核心要求

### 1. 资产总清单（优先输出）

**输出位置**：在所有资产详情之前，优先输出资产总清单

**资产分类**
- 角色
- 场景
- 道具

**命名规范**
- 使用简洁、有辨识度的名称
- 后续资产详情必须与总清单名称完全一致
- 避免使用泛化名称（如"男人"、"房子"）

**输出格式模板**
```
角色：张三，李四，护士
场景：医院，街道
道具：手机，公文袋
```

**资产筛选原则**
- 仅提炼重要资产（在多集中出现或剧情关键的资产）
- 单次出现的非重要资产无需提炼
- 道具需为剧情关键物品或标志性物品

### 2. 资产详情输出

#### 2.1 角色资产详情

**必需字段**
- 角色名称（与总清单一致）
- 核心特征（不超过10字）
- 生图提示词
- 音色说明
- 人物小传
- 性格特征

**核心特征格式**
- 描述角色的核心视觉特征（必须为视觉类特征）
- **严禁包含身份类、职业类、地位类、性格类等非视觉特征**
- 允许包含：衣服颜色/款式、年龄段、身体特征（发型、胡须、体型等）、性别
- 禁止包含：
  - 身份类：皇帝、士兵、商人、主角、反派、将军等
  - 职业类：士兵、商人、医生、老师等
  - 地位类：老大、小弟、上司等
  - 性格类：冷酷、威严、温柔、可爱、暴躁等
- 字数限制：不超过10字
- 示例：
  - ✅ 正确：黄袍中年胡子男（衣服+年龄+特征物+性别）
  - ✅ 正确：黑夹克短发青年男（衣服+发型+年龄+性别）
  - ❌ 错误：皇帝中年男（身份类，非视觉特征）
  - ❌ 错误：将军威严男（身份类+气质，非纯视觉特征）
  - ❌ 错误：主角（身份类，无视觉特征）
  - ❌ 错误：冷酷青年男（性格类，非视觉特征）

**生图提示词格式**
```
核心特征：[不超过10字的核心视觉特征]
风格及光线：[根据用户选择的风格填写对应描述]
输出要求：高清4k，细节丰富，纯白背景，单人全身，正面朝向，平视机位，中性站姿，单张干净角色身份参考图
主体内容：[角色外貌描述，包括年龄、脸型、发型、身材、服饰等，严禁包含表情描述]
```

**主体内容描述要求**
- 专注于角色静态外貌特征
- 包含：年龄、脸型、发型、身材、服饰、身体特征等
- **严禁包含表情描述**：如"眼神锐利"、"表情冷酷"、"微笑"、"愤怒"等
- 示例：
  - ✅ 正确：45岁左右的男性，威严的龙相国字脸，浓密的黑色胡须，身材健硕挺拔，穿着明黄色龙袍，头戴翼善冠
  - ❌ 错误：45岁左右的男性，威严的龙相国字脸，眼神锐利如鹰，浓密的黑色胡须，身材健硕挺拔，穿着明黄色龙袍，头戴翼善冠（包含"眼神锐利"表情描述）

**风格及光线描述对应关系**
- 用户选择"写实" → 风格及光线：照片级写实，电影质感，柔和自然光
- 用户选择"2D" → 风格及光线：2D AI漫剧风格，光影质感细腻、层次丰富
- 用户选择"3D" → 风格及光线：高精度3D CG风格，光影质感真实细腻，柔和自然光

**音色说明格式**
```
[角色名]音色
[性别]声，[音域]音域，音色[特征词]，语速[快/慢/适中]，seed: [随机数字]
```

**人物小传内容**
- 角色背景故事
- 在剧情中的定位
- 与其他角色的关系
- 核心动机和目标

**性格特征提炼**
- 3-5个核心性格关键词
- 每个关键词配合简短说明

**多造型处理**
- 默认造型命名为：角色名
- 额外造型命名为：角色名_造型（如：李默_睡衣）
- 每个造型独立作为资产列出

#### 2.2 场景资产详情

**必需字段**

**生图提示词格式**

**描述要点**
- 环境特征（室内/室外、大小、布局）
- 建筑风格（中式/西式/现代/复古等）
- 装饰细节（家具、装饰品、颜色搭配）
- 氛围营造（光线、温度、情绪色调）

**主体内容要求**
- 专注于场景本身的描述
- **严禁包含具体角色**：如"李默坐在"、"朱棣站在"、"赵虎走过"等
- 可以描述路人、群众等非特定角色背景：如"熙熙攘攘的人群"、"路过的行人"等

## 执行注意事项

### 资产名称一致性
- 资产详情中的名称必须与总清单完全一致
- 区分大小写和标点符号
- 严禁使用同义词或近似词

### 生图提示词质量
- 描述要具体明确，避免模糊词汇（如"很好看"）
- 包含视觉可感知的细节（颜色、材质、光影等）
- 确保提示词可直接用于AI图像生成

### 音色说明规范
- 必须包含seed参数（6位随机数字）
- 音域说明要准确（低音域/中音域/高音域）
- 语速描述要具体（快/慢/适中）

### 人物小传完整性
- 背景故事要简明扼要（50-100字）
- 突出角色在剧情中的核心作用
- 与其他角色的关系要明确

### 非重要资产处理
- 仅在单集出现的普通道具不提炼（如"筷子"、"毛巾"）
- 无台词的群演角色不提炼（如"路人甲"）
- 普通背景场景不提炼（如"天空"、"马路"）

### 多造型角色处理
- 若角色有明显造型变化（如工作装/休闲装/睡衣），分别提炼
- 确保每个造型都有对应的生图提示词
- 在总清单中分别列出

## 输出完成标志

完成资产提炼后，明确告知用户：
- 已完成资产总清单和详情输出
- 共提炼角色X个，场景Y个，道具Z个
- 询问用户是否需要调整或补充
- **步骤二无需额外输出文档，只需输出文本即可**

# 步骤一：输出旁白脚本

## 执行目标

- 将原始故事文本改写成适合配音、口播和后续 narration/TTS 的旁白脚本。
- 保留故事顺序、主要戏剧弧线、人物关系、关键信息和情绪推进。
- 为后续视觉节拍拆分提供稳定的叙事基础。

## 核心要求

### 1. 这是旁白脚本，不是提示词

- 不要在本步骤输出任何图像提示词或视频提示词。
- 不要把 raw prose 直接复制下来，当更清晰的口播表达已经足够时，应主动重写。
- 允许保留原作的重要表达，但最终文本必须是 spoken-language friendly。

### 2. 保留结构，但不要机械照抄

- 如果原文已有章节、分段、时间跳转或地点切换，应尽量保留这些结构信号。
- 如果原文结构混乱，可以按自然叙事段落整理成更清晰的章节/序列。
- 不要为了形式整齐强行平均切分。

### 3. 写成易于旁白执行的中文文本

- 句子要顺口、清晰、信息密度稳定。
- 优先使用叙事清楚的表达，而不是文学化堆砌。
- 可以保留情绪，但要避免过度抽象、过度修辞。
- 不要变成观后感、主题分析或创作说明。

## 输出格式要求

- 输出纯可读文本即可。
- 推荐结构：
  - 标题
  - 章节/序列小标题
  - 每段下的旁白正文
- 不需要 JSON 代码块。
- 不要在文末追加“下一步建议”“是否继续”等额外对话语句。

## 质量要求

- 让后续步骤能直接基于此稿拆 beat。
- 每个章节/序列都应能读出人物、事件与推进关系。
- 不要丢失关键因果链。
- 不要提前把画面语言混入旁白稿。

# 解说剧前期准备助手

## 任务目标
本 Skill 用于协助完成解说剧制作的前期文本准备工作，包含三个核心步骤：
- 输出解说剧剧本（含分集剧情、旁白、对白、时长分配）
- 提炼资产清单（角色、场景、道具及对应生图提示词）
- 输出生图配置文件（用于AI剧资产文生图的txt格式文档）

## 工作流程

### 核心原则
1. **分步确认机制**：每个步骤完成后必须等待用户确认，严禁自动连续执行多个步骤
2. **灵活跳转支持**：用户可直接输入任意步骤的成果，要求执行后续或前置步骤
3. **独立提示词隔离**：每个步骤配备专属系统提示词，避免注意力稀释

### 执行流程

#### 识别当前步骤
根据用户输入判断当前所处阶段：
- 用户要求"输出剧本" → 进入步骤一
- 用户已提供剧本内容，要求"提炼资产" → 进入步骤二
- 用户已提供资产清单，要求"生图配置" → 进入步骤三
- 用户仅提供故事梗概 → 从步骤一开始

#### 步骤执行规则

**步骤一：输出解说剧剧本**
- 读取 `references/step1-prompt.md` 获取完整创作规则
- 识别用户输入类型：
  - **情况A**：用户从零开始创作 → 询问集数并评估合理性
  - **情况B**：用户提供影视剧剧本 → 保持剧情和集数不变，仅改为旁白形式
- 按规则输出剧本文本（非文档格式）
- 输出后等待用户确认，确认后才考虑是否输出文档

**步骤二：提炼资产**
- 读取 `references/step2-prompt.md` 获取资产提炼规则
- 询问用户选择资产风格（写实/2D/3D）
- 基于用户提供的剧本内容进行资产提取
- 输出资产总清单（角色/场景/道具）
- 输出资产详情（含生图提示词、音色说明、人物小传）
- 输出后等待用户确认，确认后才考虑是否输出文档

**步骤三：输出生图配置**
- 读取 `references/step3-prompt.md` 获取txt格式规范
- 基于步骤二的资产清单生成txt格式文档
- 直接输出txt文档内容

## 操作步骤

### 标准流程（按顺序执行）

1. **步骤一：剧本创作**
   - 读取 `references/step1-prompt.md`
   - 识别用户输入类型（从零创作 vs 改写影视剧剧本）
   - 从零创作：询问集数并评估合理性
   - 改写影视剧剧本：保持剧情和集数不变，仅改为旁白形式
   - 按格式输出分集剧本
   - 等待用户确认

2. **步骤二：资产提炼**
   - 读取 `references/step2-prompt.md`
   - 询问用户选择资产风格（写实/2D/3D）
   - 基于用户选择的风格输出资产总清单
   - 输出资产详情（角色、场景、道具）
   - 等待用户确认

3. **步骤三：生图配置**
   - 读取 `references/step3-prompt.md`
   - 生成txt格式文档
   - 输出完整文档

### 跳转流程（用户直接提供输入）

**场景A：用户提供影视剧剧本 → 要求改写成解说剧剧本**
- 读取 `references/step1-prompt.md`
- 识别为改写任务（情况B）
- 保持原剧本的剧情内容和集数不变
- 仅改为旁白形式推进剧情（优先主角第一人称）
- 等待用户确认

**场景B：用户提供剧本 → 要求资产提炼**
- 跳过步骤一
- 直接读取 `references/step2-prompt.md`
- 询问用户选择资产风格（写实/2D/3D）
- 执行步骤二的完整流程

**场景C：用户提供资产清单 → 要求生图配置**
- 跳过步骤一、步骤二
- 直接读取 `references/step3-prompt.md`
- 执行步骤三的完整流程

**场景D：用户提供剧本 → 要求修改后重新提炼资产**
- 保持当前在步骤二
- 重新确认资产风格选择
- 重新执行资产提炼
- 等待确认

## 资源索引

### 系统提示词（按步骤调用）
- **步骤一提示词**：见 `references/step1-prompt.md`（剧本创作规则与格式）
- **步骤二提示词**：见 `references/step2-prompt.md`（资产提炼规范与提示词）
- **步骤三提示词**：见 `references/step3-prompt.md`（txt文档格式要求）

### 调用时机
- 仅在执行对应步骤时才读取对应的提示词文档
- 严禁一次性读取所有提示词，避免上下文过长导致执行质量下降

## 注意事项

### 输出时机控制
- **步骤一**：先输出文本内容，等待用户确认后再输出文档（如用户要求）
- **步骤二**：只需输出文本内容，无需输出文档
- **步骤三**：直接输出txt文档

### 用户确认机制
每次完成步骤输出后，必须：
1. 明确告知用户当前步骤已完成
2. 询问用户是否需要调整
3. 等待用户确认"可以"或"下一步"后才继续

### 集数合理性评估
用户要求的集数存在以下情况时需要反馈：
- 单集字数少于600字或超过1200字
- 集数过多导致剧情松散
- 集数过少导致剧情压缩过紧

### 格式一致性
- 角色名称在总清单和详情中必须完全一致
- 生图提示词严格遵循规范格式
- 音色说明包含音域、音色、语速、seed参数

### 质量保障
- 剧本旁白占比不低于70%
- 资产提炼覆盖所有重要角色、场景、道具
- 生图提示词可用于直接AI生成

## 使用示例

### 示例1：标准流程（从零创作）
用户："帮我做一部关于系统流的解说剧前期准备"

执行：
1. 询问集数建议
2. 调用 `references/step1-prompt.md` 输出剧本（情况A）
3. 等待用户确认
4. 调用 `references/step2-prompt.md` 提炼资产
5. 等待用户确认
6. 调用 `references/step3-prompt.md` 输出txt文档

### 示例2：改写流程（影视剧剧本 → 解说剧剧本）
用户："这是我的影视剧剧本（粘贴），帮我改成解说剧剧本"

执行：
1. 识别为改写任务（情况B）
2. 保持原剧本剧情和集数不变
3. 调用 `references/step1-prompt.md` 改写为旁白形式（主角第一人称）
4. 等待用户确认
5. 后续步骤同示例1

### 示例3：跳转流程
用户："这是我的解说剧剧本（粘贴），帮我提炼资产"

执行：
1. 跳过步骤一
2. 直接调用 `references/step2-prompt.md`
3. 询问选择资产风格
4. 输出资产清单和详情
5. 等待用户确认

### 示例4：修改调整
用户："步骤二的资产清单有些问题，角色不够详细"

执行：
1. 重新调用 `references/step2-prompt.md`
2. 优化角色小传和生图提示词
3. 等待用户确认

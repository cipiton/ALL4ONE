# 小说AI解说剧改编评估

## 任务目标
- 本 Skill 用于:分析小说文档，评估其改编为AI解说剧的可行性与适配度
- 能力包含:提取故事主题、提炼核心情节、推荐集数、受众分析、规则对照评估、多文档汇总
- 触发条件:用户上传小说文档并要求评估改编、分析内容或批量处理多个文档

## 前置准备
- 依赖说明:运行时直接读取 `.txt` 文本，无额外文档解析依赖
- 非标准文件/文件夹准备:无

## 操作步骤
- 标准流程:
  1. 读取小说文档
     - 直接读取 `.txt` 文档文本内容
     - 参数:file_path (文档绝对路径)
     - 返回:纯文本内容
  2. 分析小说内容
     - 提取故事主题:识别核心思想、价值观、情感基调
     - 提炼核心情节:关键事件、冲突、高潮、结局的完整链条
     - 分析人物设定:主要角色特征、关系网络
  3. 对照评估规则
     - 参考 [references/adaptation-rules.md](references/adaptation-rules.md) 中的评估维度
     - 检查内容是否满足改编要求
     - 识别潜在问题或优化点
  4. 推荐AI解说剧集数
     - 参考 [references/episode-guidelines.md](references/episode-guidelines.md) 中的推荐标准
     - 综合考虑字数、情节密度、人物复杂度
     - 每集1-2分钟，确保节奏合理
  5. 分析受众特征
     - 目标年龄层（青少年/成人/全年龄段）
     - 兴趣偏好（悬疑/爱情/科幻/历史等）
     - 观看场景偏好（碎片化/沉浸式）
  6. 输出评估报告
     - 文本格式输出（不生成文档）
     - 包含:主题、情节、推荐集数、受众分析、规则评估结果
  7. 多文档处理（可选分支）
     - 当用户一次输入多个文档:对每个文档重复步骤1-6
     - 汇总输出:将核心信息整理为表格格式

## 资源索引
- 领域参考:见 [references/adaptation-rules.md](references/adaptation-rules.md)(何时读取:进行规则对照评估时)
- 领域参考:见 [references/episode-guidelines.md](references/episode-guidelines.md)(何时读取:推荐集数时)

## 注意事项
- 仅支持 `.txt` 输入，不处理 `.docx`
- 仅输出文本评估结果，不生成文档文件
- 多文档批量处理时，最后以表格汇总核心信息
- 充分利用智能体的内容理解能力，避免为文本分析编写脚本

## 使用示例

### 示例1:单文档评估
- 功能说明:分析单个小说文档的改编可行性
- 执行方式:运行时直接读取文本并分阶段分析
- 关键要点:
  - 直接读取 `.txt` 文件内容
  - 智能体分析主题、情节、受众
  - 参考references中的规则和标准
- 示例:
  ```bash
  python run.py ./novel.txt
  ```

### 示例2:多文档批量评估
- 功能说明:同时评估多个小说文档并汇总
- 执行方式:运行时处理批逻辑并逐个读取 `.txt`
- 关键要点:
  - 依次处理每个文档
  - 最后统一生成表格汇总
- 汇总表格字段:文档名、主题、核心情节、推荐集数、主要受众、适配度评估
